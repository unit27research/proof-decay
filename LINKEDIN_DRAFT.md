# LinkedIn Draft

Field note from another small local instrument in the evidence-boundary lane.

Humility Engine asks whether a claim outruns its evidence.

Claim Drift asks whether a claim became stronger between drafts.

Proof Decay asks whether old proof is being reused as if it still supports a current claim.

The pattern is common:

- A demo ran six months ago.
- A screenshot still exists.
- The code changed.
- The claim now says the workflow is ready for public release.

The artifact may be real. That does not mean it is still current proof.

Proof Decay is intentionally modest: a local CLI that reads current claims and structured proof notes, then flags possible stale artifacts, changed conditions, and proofwashing risks for human review.

It does not verify truth. It does not prevent fraud. It does not certify readiness, compliance, safety, or factual accuracy.

It is a heuristic review aid: proof before claim, current proof before current claim.
